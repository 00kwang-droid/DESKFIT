# 데스크핏 · 자리운동일지 (DeskFit)

자리에 앉은 채로, 티 안 나게 하는 직장인용 등척성 운동 PWA입니다.

## 실행 방법 (테스트)

```bash
cd deskfit
python3 -m http.server 8000
# http://localhost:8000 접속
```

**주의:** `file://`로 더블클릭해서 열면 Service Worker(알림·오프라인 캐시)가 동작하지 않습니다. 반드시 http(s) 서버로 열어야 합니다.

## 실제 배포

GitHub Pages, Netlify, Vercel 등 정적 호스팅에 폴더 그대로 올리면 됩니다(HTTPS 필수). 아이콘은 루트에 두면 되고 별도 `icons/` 폴더가 필요 없습니다. 휴대폰에서 "홈 화면에 추가"하면 앱처럼 실행됩니다.

## 폴더 구조

```
deskfit/
├── index.html      화면 구조 (오늘/리포트/설정/타이머/온보딩/부위선택)
├── styles.css      다크 에디토리얼 톤 디자인 시스템
├── app.js          부위 선택, 스케줄 생성, 타이머, 알림, 칼로리, 리포트 로직
├── sw.js           오프라인 캐시 + 알림 클릭 라우팅
├── manifest.json   PWA 설치 메타데이터
├── icon-192.png
└── icon-512.png
```

## 되는 것

- **최초 1회 온보딩**: 출근/퇴근/점심시간, 운동 간격(45/60/75/90분), 몸무게(칼로리 계산용), 강도 설정
- **매일 아침 부위 선택**: 대분류(상체·코어·하체)와 중분류(어깨·가슴·등·팔·목 / 복부·옆구리·허리·골반저 / 허벅지·엉덩이·종아리·발목)를 골라 그 부위 위주로 하루 스케줄 자동 생성. 안 고르면 전신 밸런스.
- **다양한 운동**: 부위별 여러 종목이 매일 다르게(셔플 + 부위 순환) 배치됩니다.
- **오늘 화면**: 시간순 카드(부위·소요시간·예상 칼로리) + 진행률 + 연속 실행일
- **타이머**: 원형 카운트다운을 중심으로 위=운동 방법, 아래=이 운동이 어디에 도움이 되는지 표시. 완료 시 자동 체크, 스킵 시 사유 선택.
- **리포트**:
  - 오늘 요약 — 완료 개수, 오늘 소모 칼로리, 자동 평가 문구
  - **클로드에게 오늘 운동 평가받기** 버튼 → 오늘 기록을 프롬프트로 담아 claude.ai를 새 탭으로 열어 코칭 평가를 받습니다
  - 이번 주 요약 — 실행률, 주간 소모 칼로리, 최고 연속, 잘 지킨 시간대
  - 요일별 실행률 그래프 + 인사이트
- **설정**: 근무시간/점심/간격/몸무게/강도/알림, 부위 다시 골라 재생성
- 전부 로컬 저장(localStorage), 오프라인 지원

## 칼로리 계산

`kcal = MET × 3.5 × 체중(kg) / 200 × 운동시간(분)` 공식으로 운동별 MET(1.9~3.0)를 적용해 완료한 운동의 칼로리를 합산합니다. 등척성 위주라 하루 수치는 크지 않지만(수십 kcal대) 실제에 가깝게 정직하게 표시합니다.

## 클로드 평가 방식

배포된 정적 PWA에서는 API 키를 클라이언트에 넣기 어렵고 노출 위험이 있어, 테니스일지와 동일하게 **claude.ai 딥링크** 방식으로 구현했습니다. 오늘의 완료/스킵/칼로리/부위를 요약한 프롬프트를 만들어 `https://claude.ai/new?q=...`로 여는 방식이라, 어디에 배포하든 별도 서버 없이 동작합니다.

## 알아두셔야 할 한계 (알림)

브라우저 로컬 알림은 앱(탭)이 완전히 종료되면 정시 알림이 안 갈 수 있습니다. 열어둔 상태에서는 잘 동작하지만, 완전한 백그라운드 푸시는 Firebase Cloud Messaging(FCM) 연동이 필요합니다.

## 다음에 붙이면 좋은 것

1. **Firebase 연동** — 기기 변경 시 데이터 유지, 진짜 백그라운드 푸시
2. **운동 즐겨찾기/커스텀** — 자주 하는 종목 고정, 사용자 운동 추가
3. **주간 목표** — "이번 주 실행률 70%" 같은 목표 대비 진행
4. **부위별 누적 통계** — 어디를 많이/적게 했는지 밸런스 리포트

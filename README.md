# 데스크핏 · 자리운동일지 (DeskFit)

자리에 앉은 채로, 티 안 나게 하는 직장인용 등척성 운동 PWA입니다.

## 실행 방법 (테스트)

정적 파일이라 아무 웹 서버로 열면 됩니다. 로컬에서 확인하려면:

```bash
cd deskfit
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

**주의:** `file://`로 더블클릭해서 열면 Service Worker(알림·오프라인 캐시)가 동작하지 않습니다. 반드시 http(s) 서버를 통해 열어야 합니다.

## 실제 배포

GitHub Pages, Netlify, Vercel 등 정적 호스팅에 폴더 그대로 올리면 됩니다. HTTPS가 필수입니다 (알림 권한은 HTTPS/localhost에서만 동작).
휴대폰에서 접속 후 "홈 화면에 추가"를 하면 앱처럼 아이콘이 생기고 standalone 모드로 실행됩니다.

## 폴더 구조

```
deskfit/
├── index.html      화면 구조 (오늘/리포트/설정/타이머/온보딩)
├── styles.css       다크 에디토리얼 톤 디자인 시스템
├── app.js           스케줄 생성, 타이머, 알림, 리포트 로직 전체
├── sw.js             오프라인 캐시 + 알림 클릭 라우팅
├── manifest.json    PWA 설치 메타데이터
└── icons/           앱 아이콘 (192/512)
```

## 지금 되는 것

- 온보딩에서 출근/퇴근/점심시간 + 운동 강도(완전히 안 보임 / 약간 허용)를 입력하면 하루 스케줄 자동 생성
- 오늘 화면에서 시간순 운동 카드 + 진행률 바 + 연속 실행일(스트릭)
- 카드 탭 → 원형 타이머 실행 → 완료 시 자동 체크, 스킵 시 사유 선택
- 2분 전 준비 알림 + 정시 시작 알림 (탭하면 바로 해당 운동 타이머로 진입)
- 리포트 탭에서 주간 실행률, 요일별 그래프, 가장 잘 지킨 시간대 인사이트
- 설정에서 근무시간/점심시간/강도/알림 on-off, 오늘 일정 재생성
- 전부 로컬 저장(localStorage)이라 서버 없이 완전히 동작 + 오프라인 지원

## 알아두셔야 할 한계 (중요)

브라우저의 알림 스케줄링은 **앱(탭)이 완전히 종료되면 정시 알림이 안 갈 수 있습니다.** 지금 구현은 앱을 열어둔 상태(백그라운드 탭 포함)에서는 잘 동작하지만, 아이폰/안드로이드에서 브라우저를 완전히 꺼버리면 알림이 안 옵니다.

**완전한 백그라운드 알림**을 원하시면 다음 단계로 확장이 필요합니다:
- Firebase Cloud Messaging(FCM) 연동 → 서버에서 정시에 Push 발송 (이미 다른 프로젝트에서 Firebase Firestore 쓰고 계시니 같은 프로젝트에 붙이면 됨)
- 또는 iOS는 홈 화면에 추가한 PWA + iOS 16.4 이상에서 Web Push 지원하므로 FCM으로 커버 가능

지금 버전은 "책상에 앉아 앱을 열어두거나 자주 확인하는" 실사용 패턴에서는 충분히 잘 동작하고, 다음 단계로 넘어갈 좋은 기반이 됩니다.

## 다음에 붙이면 좋은 것

1. **Firebase 연동** — 기기 변경 시에도 데이터 유지, 진짜 백그라운드 푸시
2. **가디언/공유 기능** — 테니스일지처럼 배우자나 동료와 오늘의 실행률 공유
3. **운동 커스텀** — 사용자가 직접 운동을 추가/편집
4. **주간 목표 설정** — "이번주 실행률 70% 이상" 같은 목표 대비 진행 표시
